# Chisinau Passport E-Queue Telegram Bot

Бот каждые 2 минуты проверяет страницу электронной очереди ДП «Документ» в Кишинёве и отправляет Telegram-уведомление, если страница больше не выглядит как «все места заняты» и появляются признаки формы записи.

Новая версия поддерживает подписку: уведомления получают все пользователи или чаты, которые отправили боту `/start`. Команда `/stop` отключает уведомления для конкретного чата. После `/start` бот показывает кнопку **«🔎 Проверить статус сейчас»** — по ней пользователь получает текущий статус очереди, не дожидаясь очередной автоматической проверки.

Целевая страница:

```txt
https://chisinau.pasport.org.ua/solutions/e-queue
```

## Что именно отслеживается

Бот считает, что мест нет, если в тексте страницы найдены фразы вроде:

- `Наразі всі місця зайняті`
- `всі місця зайняті`
- `відсутні місця`
- `немає вільних місць`
- `спробуйте в інший час або день`

Бот считает, что возможно появились места, если отрицательных фраз нет и одновременно видны признаки формы:

- `Обрати день`
- `Обрати час`
- `Номер телефону`

Это не автозапись. Бот только присылает пуш, после чего страницу нужно открыть вручную и пройти верификацию.

## Как работает подписка

1. Пользователь открывает вашего Telegram-бота.
2. Отправляет `/start`.
3. Бот сохраняет `chat.id` в файл подписчиков.
4. Когда появляются признаки окон, бот делает рассылку всем подписчикам.
5. Если пользователь нажимает кнопку **«🔎 Проверить статус сейчас»** или отправляет `/status`, бот делает свежую проверку страницы и отвечает только этому пользователю.
6. Если пользователь отправляет `/stop`, его `chat.id` удаляется из списка.

Список подписчиков хранится в JSON-файле. Для Railway желательно использовать Volume и путь вроде `/data/subscribers.json`, чтобы подписчики не потерялись при redeploy/restart.

## Локальный запуск

```bash
npm install
cp .env.example .env
```

Заполнить `.env`:

```env
TELEGRAM_BOT_TOKEN=123456789:replace_me
CHECK_INTERVAL_MINUTES=2
SUBSCRIBERS_FILE=./data/subscribers.json
```

`TELEGRAM_CHAT_ID` теперь не обязателен. Его можно оставить только как начального подписчика, чтобы при первом запуске автоматически добавить себя в список.

Проверить один раз:

```bash
npm run check:once
```

Запустить в режиме постоянного мониторинга:

```bash
npm run dev
```

Production-сборка:

```bash
npm run build
npm run start
```

## Как получить Telegram Bot Token

1. Открыть Telegram-бота `@BotFather`.
2. Отправить `/newbot`.
3. Задать имя и username бота.
4. Скопировать токен в `TELEGRAM_BOT_TOKEN`.

## Как подписаться и проверить статус вручную

После запуска бота в Railway или локально:

1. Откройте своего бота в Telegram.
2. Нажмите **Start** или отправьте `/start`.
3. Бот ответит, что подписка включена, и покажет кнопки:
   - **🔎 Проверить статус сейчас** — сделать свежую проверку страницы;
   - **🌐 Открыть страницу записи** — перейти на сайт вручную.
4. Команда `/status` делает то же самое, что кнопка проверки статуса.

Для группы: добавьте бота в группу и отправьте `/start` в группе. После этого уведомления будут приходить в группу, а кнопка проверки статуса будет работать в этом чате.

## Переменные окружения

| Переменная | Обязательная | По умолчанию | Описание |
|---|---:|---:|---|
| `TELEGRAM_BOT_TOKEN` | да | — | Токен от BotFather |
| `TELEGRAM_CHAT_ID` | нет | — | Необязательный начальный подписчик |
| `CHECK_URL` | нет | `https://chisinau.pasport.org.ua/solutions/e-queue` | URL страницы очереди |
| `CHECK_INTERVAL_MINUTES` | нет | `2` | Интервал проверки |
| `ALERT_COOLDOWN_MINUTES` | нет | `30` | Минимальный перерыв между повторными уведомлениями |
| `CHECK_TIMEOUT_MS` | нет | `20000` | Таймаут запроса к сайту |
| `STARTUP_NOTIFY` | нет | `false` | Присылать сообщение подписчикам при старте бота |
| `NOTIFY_ON_UNKNOWN` | нет | `false` | Уведомлять, если статус страницы не распознан |
| `LOG_HTML_SNIPPET_ON_UNKNOWN` | нет | `false` | Печатать кусок текста страницы в логах при `UNKNOWN` |
| `TELEGRAM_POLLING_INTERVAL_SECONDS` | нет | `5` | Как часто бот проверяет команды `/start`, `/stop`, `/status` и нажатия кнопок |
| `SUBSCRIBERS_FILE` | нет | `./data/subscribers.json` | Путь к файлу подписчиков |
| `USER_AGENT` | нет | свой | User-Agent для запроса |

## Railway deploy

1. Создать GitHub-репозиторий и залить туда проект.
2. В Railway нажать **New Project** → **Deploy from GitHub repo**.
3. Выбрать репозиторий.
4. В сервисе открыть **Variables** и добавить:

```env
TELEGRAM_BOT_TOKEN=...
CHECK_INTERVAL_MINUTES=2
ALERT_COOLDOWN_MINUTES=30
STARTUP_NOTIFY=true
TELEGRAM_POLLING_INTERVAL_SECONDS=5
SUBSCRIBERS_FILE=/data/subscribers.json
```

5. Желательно подключить Railway Volume и смонтировать его в `/data`. Тогда файл `/data/subscribers.json` будет сохраняться между перезапусками и redeploy.
6. Railway сам соберёт Node.js-проект. В `railway.json` явно заданы команды сборки и запуска, чтобы не получить ошибку вида `Cannot find module dist/index.js`:

```bash
npm run build
npm run start
```

## Почему это worker, а не cron

Для интервала 2 минуты лучше постоянный background worker:

- состояние последнего статуса хранится в памяти процесса;
- уведомление отправляется только при смене статуса или после cooldown;
- бот параллельно слушает команды `/start`, `/stop`, `/status` и нажатия кнопок;
- процесс не должен завершаться после одной проверки.

Railway Cron больше подходит для коротких задач, которые стартуют по расписанию и завершаются. Здесь удобнее держать один живой процесс.

## Настройка частоты

Сейчас стоит:

```env
CHECK_INTERVAL_MINUTES=2
```

Если сайт начнёт ограничивать частые запросы, можно быстро поставить 5 или 10 минут без изменения кода.

## Возможные статусы в логах

```txt
NO_SLOTS        страница явно пишет, что мест нет
POSSIBLE_SLOTS  страница похожа на открытую форму записи
UNKNOWN         сайт изменился или ответ не похож ни на один ожидаемый вариант
ERROR           HTTP-ошибка, таймаут или сетевой сбой
```

## Важное ограничение

Бот не проходит Дію/BankID и не бронирует слот. Он только сообщает, что страница изменилась и, вероятно, появилась возможность записаться.
